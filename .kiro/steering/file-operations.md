# 文件操作安全规则

## 删除或替换文件前必须确认目标已就绪

在执行任何删除、移动、重命名操作之前，必须满足以下条件之一：

1. **新文件已创建完成** — 替换场景下，先用 `fsWrite` 或 `fsAppend` 完整写入新文件，确认文件存在且内容正确后，再删除旧文件。
2. **不需要替换** — 纯删除场景下，明确确认该文件不再被任何地方引用。

## 禁止的操作顺序

❌ 错误顺序：先删除旧文件 → 再创建新文件（中间窗口期文件缺失）

✅ 正确顺序：先创建新文件并验证 → 再删除旧文件

## 适用场景举例

- 重构/重命名文件：先写新路径的文件，确认写入成功，再删旧文件
- 拆分文件：先创建所有拆分后的文件，再删除原文件
- 合并文件：先创建合并后的文件，再删除被合并的文件
- 优化/改写：先用 `strReplace` 或 `fsWrite` 完成改写，再做任何清理

## 验证步骤

删除前执行 `getDiagnostics` 或 `go build` 确认新文件编译无误，再执行删除。
